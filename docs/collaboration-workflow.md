# Collaboration Workflow

IRE is designed for advisor-led research and student group collaboration. The workflow should make progress visible, reproducible, and reusable.

## Advisor Responsibilities

- Define the research vision, target venues, and quality bar.
- Align group modules with long-term ChatREQ and IRE-Bench goals.
- Review research questions, methodology, baselines, and evaluation protocols.
- Provide feedback on weekly reports, experiment results, and paper drafts.
- Ensure that datasets, knowledge assets, and benchmark artifacts remain reusable.

## Student Responsibilities

- Maintain weekly progress reports.
- Keep group artifacts organized in the assigned folders.
- Document datasets, prompts, system modules, experiments, and results.
- Use GitHub issues and pull requests for visible collaboration.
- Connect implementation work to research questions and paper claims.
- Prepare final deliverables that can be reused by future cohorts.

## Codex Responsibilities

Codex can support the project as a research software engineering assistant. Its responsibilities include:

- Scaffolding repository structure and documentation.
- Generating paper skeletons and section drafts.
- Refactoring system modules while preserving existing work.
- Creating experiment scripts, result tables, and reproducibility checklists.
- Reviewing consistency across README files, benchmarks, datasets, and papers.
- Helping students turn weekly progress into paper-ready artifacts.

Codex output should always be reviewed by humans before publication or release.

## GitHub Workflow

Use GitHub as the shared record of research progress:

- Issues track tasks, bugs, research questions, dataset gaps, and paper TODOs.
- Branches isolate group or module work.
- Pull requests provide review points for system, data, benchmark, and paper changes.
- Commits should be small and descriptive.
- Releases can mark benchmark, dataset, and paper submission milestones.

## Weekly Reports

Weekly reports should include:

- Completed work.
- Current blockers.
- Dataset or benchmark updates.
- Experiments run and key results.
- Figures, tables, or paper text produced.
- Next week plan.

## Benchmark Updates

Benchmark changes should document:

- Task definition.
- Input and output format.
- Dataset split.
- Baselines.
- Metrics.
- Evaluation scripts.
- Known limitations.

## Paper Review Workflow

Paper drafts should be reviewed for:

- Clarity of research questions.
- Strength of motivation.
- Correctness of method description.
- Quality of baselines and ablations.
- Statistical and practical significance of results.
- Threats to validity.
- Reproducibility.
- Alignment with ACM SIGCONF style.

## Dataset Maintenance

Dataset maintainers should track:

- Source and license.
- Schema version.
- Annotation guidelines.
- Annotator agreement or quality control.
- Processing scripts.
- Sensitive data review.
- Benchmark compatibility.
