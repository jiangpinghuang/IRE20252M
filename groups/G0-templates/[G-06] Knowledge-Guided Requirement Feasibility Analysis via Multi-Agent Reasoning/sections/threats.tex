\section{Threats to Validity}
\label{threats}

\subsection{Internal Validity}
\label{threats:internal}

Internal validity threats include annotation bias, inconsistent human feasibility judgments, prompt sensitivity, and possible leakage between knowledge sources and evaluation examples.
The study should mitigate these risks through clear annotation guidelines, independent review, and prompt/version reporting.

\subsection{External Validity}
\label{threats:external}

External validity threats concern whether results generalize across domains, requirement styles, project sizes, and deployment environments.
The evaluation should therefore report domain coverage and avoid claiming generality beyond the evaluated requirement corpus.

\subsection{Construct Validity}
\label{threats:construct}

Construct validity threats arise if feasibility, realism, or human review reduction are measured too narrowly.
The metric design should align with the research questions and distinguish feasibility problems from broader requirement quality issues.
