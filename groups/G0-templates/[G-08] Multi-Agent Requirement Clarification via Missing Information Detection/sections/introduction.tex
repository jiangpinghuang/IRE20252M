\section{Introduction}
\label{introduction}

\subsection{Background and Motivation}
\label{sec:intro-background}

Natural language remains the dominant form for expressing software requirements, yet stakeholder-readable statements are rarely directly executable by intelligent software agents.
Recent work has emphasized requirements as a central interface for AI-driven software engineering~\cite{robinsonRequirementsAreAll2025}, but moving from textual intent to executable behavior is still an open problem.
This paper focuses on \emph{executable requirements}: requirements that can be mapped to explicit tasks, actions, dependencies, and execution workflows.

\subsection{Challenges of Executable Requirements}
\label{sec:intro-challenges}

Natural language requirements often omit operational semantics.
They describe expected capabilities but do not specify which actions should be performed, how tasks should be ordered, or how constraints affect execution.
LLM-based requirement generation can improve textual coverage, but generated requirements may still lack behavior mappings and task-level structure.
This creates a gap between requirement engineering artifacts and agent-executable workflows.

\subsection{Multi-Agent Task Planning}
\label{sec:intro-planning}

We study whether role-based agent collaboration can support task transformation.
Instead of treating requirement analysis as a single text generation step, the proposed workflow separates semantic analysis, task planning, and execution-flow checking across specialized agents.
Knowledge-guided analysis provides domain concepts, task templates, and planning rules that constrain the transformation process.

\subsection{Contributions}
\label{sec:intro-contributions}

This paper makes the following contributions:
\begin{itemize}[leftmargin=*]
  \item We formulate executable requirements as a research problem in intelligent requirements engineering.
  \item We propose a knowledge-guided multi-agent workflow for transforming natural language requirements into actionable task representations.
  \item We define an executable requirement construction process covering task decomposition, action sequencing, requirement-to-task mapping, and execution-flow construction.
  \item We design an evaluation plan for measuring task generation accuracy, execution completeness, requirement-to-task alignment, and planning consistency.
\end{itemize}
