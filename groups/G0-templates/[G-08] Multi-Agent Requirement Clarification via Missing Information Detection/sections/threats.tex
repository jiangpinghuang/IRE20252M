\section{Threats to Validity}
\label{threats}

\subsection{Internal Validity}
\label{sec:threats-internal}

Internal validity threats mainly arise from annotation subjectivity and evaluator bias when judging whether a generated task is correct or complete.
The study should mitigate these threats through explicit annotation guidelines, independent review, and disagreement resolution.

\subsection{External Validity}
\label{sec:threats-external}

External validity threats concern whether the findings generalize beyond the selected requirements and software domains.
The evaluation should include requirements with different functional intents, constraint types, and workflow complexity, while avoiding claims beyond the studied dataset.

\subsection{Construct Validity}
\label{sec:threats-construct}

Construct validity threats arise if the metrics do not fully capture executability.
Task accuracy, execution completeness, alignment, and consistency should therefore be interpreted together rather than as isolated indicators.
