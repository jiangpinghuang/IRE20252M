\begin{abstract}
Natural language requirements often describe expected functionality while leaving deployment environments, device constraints, runtime conditions, and user scenarios implicit.
This paper studies how multi-agent collaboration and knowledge-assisted context understanding can improve environment-aware requirement elicitation.
The planned approach focuses on a lightweight elicitation workflow in which agents identify missing environmental assumptions, refine scenario context, and involve human stakeholders when clarification is needed.
The evaluation is organized around context completeness, environment consistency, requirement completeness, and clarification effectiveness.
\end{abstract}
