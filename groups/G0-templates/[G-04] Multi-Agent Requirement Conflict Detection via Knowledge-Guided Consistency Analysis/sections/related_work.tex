\section{Related Work}
\label{related_work}

\subsection{Requirement Conflict Detection}
This line of work studies how inconsistent or incompatible requirements can be
identified before they affect downstream development. The discussion will focus on
conflict types, detection assumptions, and the extent to which existing methods
handle implicit conflicts in natural language specifications.

\subsection{Requirement Consistency Analysis and Validation}
Requirement consistency analysis examines whether requirement statements can hold
together under shared system goals, constraints, and stakeholder expectations.
Requirement validation complements this by checking whether reported conflicts are
meaningful to stakeholders and actionable for requirement refinement.

\subsection{LLM-based Requirement Analysis}
LLM-based requirement analysis has been used for requirement extraction,
classification, summarization, and validation. This paper only considers such work
where it is directly relevant to requirement consistency verification and conflict
detection, rather than general language model capabilities.

\subsection{Knowledge-Guided Requirement Verification}
Knowledge-guided verification uses domain concepts, requirement patterns, constraint
rules, and consistency criteria to reduce purely text-similarity-based judgments.
This direction is relevant because many conflicts depend on unstated assumptions or
domain-specific constraints.

\subsection{Multi-Agent Collaboration}
Multi-agent collaboration can decompose requirement verification into complementary
roles, such as analysis, validation, and review. The related work discussion will
focus on how role separation can improve conflict detection reliability without
turning the paper into a general agent platform study.
