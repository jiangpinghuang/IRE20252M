# groups

本目录用于组织 8 个研究小组的协作材料。每个小组使用统一目录结构，便于课程管理、横向比较和后续归档。

## 研究方向

- `G1-elicitation/`：需求获取与访谈、对话、用户输入分析。
- `G2-environment/`：环境建模、上下文识别与运行约束分析。
- `G3-req-graph/`：需求图谱、需求关系建模与结构化表示。
- `G4-conflict/`：需求冲突检测、解释与协调。
- `G5-hitl/`：人在回路的需求工程流程与交互机制。
- `G6-feasibility/`：需求可行性分析、风险评估与证据支持。
- `G7-architecture/`：需求到架构的映射、设计决策与一致性分析。
- `G8-executable/`：可执行需求、仿真、验证与自动化检查。

## 统一目录结构

各组目录下统一使用：

- `paper/`：小组论文工作区。
- `reports/`：阶段报告和汇报材料。
- `artifacts/`：研究产物、标注规范和中间材料。
- `experiments/`：实验设计、脚本入口和运行记录。
- `datasets/`：数据集说明、索引或可共享数据。
- `figures/`：图表和可复用图片素材。
- `src/`：代码和工具实现。
- `notes/`：会议记录、阅读笔记和想法记录。

## 提交规范

提交应聚焦单一主题，提交信息简洁说明变更内容。不要提交临时文件、缓存文件、个人密钥、未授权数据或大体积中间产物。跨组共享材料应优先放入公共目录，并在小组目录中引用。
